# Introduction

## Histoire d'origine
CSPro est l'outil de recensement et d'enquête le plus largement utilisé dans la plupart des pays africains. Il est choisi par beaucoup précisément parce qu'il offre une grande flexibilité dans la préparation des questionnaires et dans la manière dont les données sont synchronisées depuis le terrain.

Cependant, pendant le travail de terrain, il stocke les données reçues dans un format qui n'est pas facilement accessible et interrogeable. Si vous souhaitez obtenir des informations sur les données collectées, il existe peu de moyens pour y parvenir.

Notre première tentative pour résoudre ce problème consistait à "extraire" les données dans un format de base de données relationnelle en utilisant un outil que nous avons créé appelé [pyCSPro](https://pypi.org/project/pycspro/). Plus tard, CSPro a ajouté un outil similaire à leur suite CSWeb (la partie serveur de réception des données de terrain de CSPro) et a rendu une version relationnelle des données disponible.

Alors que les premières versions de nos tableaux de bord étaient spécifiquement construites et dépendaient de la base de données extraite, les versions actuelles peuvent être utilisées avec presque tout type de source de données moderne.

## Philosophie
Notre philosophie directrice dans la conception de cet outil est de rendre la création de tableaux de bord pour les exercices de terrain tels que les recensements et les enquêtes aussi simple et directe que possible.

Nous nous soucions de tous les cas particuliers et des détails afin que vous n'ayez pas à le faire. Vous pouvez simplement vous concentrer sur les indicateurs que vous souhaitez avoir et le kit s'occupera du reste.

## Utilisations
Nous envisageons que le Dashboard Starter Kit soit utilisé dans toutes sortes de scénarios où il est nécessaire d'obtenir des informations sur des données. Il s'agirait généralement d'indices de performance clés et d'autres indicateurs permettant de surveiller la qualité des données.
